# وصايا عرض مشروع التخرج - منصة تالنت هب

## قبل العرض (يوم العرض)

- [ ] تأكد أن الجهاز متصل بالكهرباء
- [ ] أوقف تشغيل الإشعارات (وضع عدم الإزعاج)
- [ ] أغلق كل البرامج غير الضرورية
- [ ] شغّل المشروع بالضغط مرتين على ملف `run-project.bat`
- [ ] تأكد أن المتصفح فتح صفحة تسجيل الدخول
- [ ] تأكد أن الخادم يعمل بفتح الرابط: `http://localhost:4000/api/health` ويجب أن يظهر: `{"status":"ok"}`

---

## سكربت الكلام (٥ دقائق)

### الافتتاح (٣٠ ثانية)
> "مشروعنا هو منصة **تالنت هب**، نظام متكامل لإدارة فرص العمل والمهارات، يربط بين الطلاب والشركات والجامعات تحت إشراف المسؤول. المنصة مبنية بتقنيات حديثة وتدعم اللغة العربية بالكامل."

---

### المعمارية والتقنيات (٦٠ ثانية)
> "المشروع يتكون من جزأين رئيسيين:
> - **الواجهة الأمامية**: صفحات ويب بلغة هيتمل وسي إس إس مع مكتبة بوتستراب ٥ بدعم كامل للغة العربية من اليمين لليسار
> - **الواجهة الخلفية**: خادم مبني بتقنية نود جي إس وإكسبريس مع قاعدة بيانات عبر طبقة بريزما
>
> نظام المصادقة يعتمد على رموز الوصول المشفّرة مع أربعة أدوار: مسؤول، طالب، شركة، جامعة.
> كذلك يوجد خيار بديل بلغة بي إتش بي لمن يفضّل بيئة زامب."

---

### العرض الحي (٩٠ ثانية)
> "دعونا نرى المشروع مباشرةً."

**الخطوات التي تنفّذها أثناء الكلام:**
1. افتح صفحة تسجيل الدخول على المتصفح
2. قل: *"هذه صفحة تسجيل الدخول، تدعم أدواراً مختلفة لكل نوع من المستخدمين"*
3. سجّل دخولاً بحساب المسؤول: البريد `admin@talenthub.local` وكلمة المرور `Admin1234`
4. قل: *"بعد التحقق من الصلاحية، يتم التوجيه تلقائياً للوحة التحكم الخاصة بالدور"*
5. استعرض لوحة التحكم الإدارية (الطلاب، الشركات، الجامعات، الوظائف، المهارات)
6. قل: *"المسؤول يستطيع إدارة كل كيانات النظام من مكان واحد"*
7. افتح نافذة جديدة وسجّل دخول كطالب، أرِ صفحة الوظائف وصفحة ملفه الشخصي

---

### جودة الكود (٦٠ ثانية)
> "على صعيد جودة الكود:
> - نفّذنا أداة **فحص الكود التلقائي** للتحقق من جودته وسلامته
> - كتبنا **اختبارات وحدة واختبارات تكامل** وجميعها تنجح بنجاح
> - كتبنا **اختبارات شاملة** تحاكي المستخدم الحقيقي على المتصفح
> - أضفنا **نظام تكامل مستمر** يُشغّل جميع الاختبارات تلقائياً عند كل تحديث
> - نتيجة الفحص الأمني: **صفر ثغرات**"

---

### الخاتمة (٣٠ ثانية)
> "المشروع جاهز للنشر على السحابة عبر منصة ريندر كما هو موضح في ملف الإعداد.
> نقدر نلخص بأن منصة تالنت هب متكاملة، مختبرة، آمنة، وجاهزة للاستخدام الفعلي."

---

## نقاط القوة التي يجب إبرازها

| النقطة | الدليل |
|--------|--------|
| أمان المصادقة | رموز وصول وتجديد مشفّرة، كلمات مرور مُعالَجة بخوارزمية آمنة |
| جودة الكود | فحص تلقائي للكود + ٦ اختبارات تنجح بالكامل |
| التوثيق | ملف التعريف + دليل الإعداد + ملف النشر السحابي |
| قابلية التوسع | ٤ أدوار مختلفة، بنية معمارية واضحة ومنظّمة |
| سهولة التشغيل | ملف تشغيل واحد يُشغّل كل شيء بضغطة |

## في حال حدوث مشكلة أثناء العرض

### إذا لم يفتح المتصفح تلقائياً
```
افتح المتصفح يدوياً واذهب إلى:
http://127.0.0.1:5500/login/login.html
```

### إذا ظهر خطأ في تسجيل الدخول
```
افتح الرابط: http://localhost:4000/api/health
إذا لم يستجب → افتح سطر الأوامر وشغّل:
  cd backend
  npm run dev:api
```

### إذا كان المنفذ مشغولاً
```
في سطر الأوامر (باور شيل):
  netstat -ano | findstr :4000
  taskkill /PID <الرقم الذي ظهر> /F
```

### إعادة تهيئة قاعدة البيانات (في حالات الطوارئ)
```
npm --prefix backend run prisma:push
npm --prefix backend run seed
```
