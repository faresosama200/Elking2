ما هي منصة تالنت هب؟

منصة تالنت هب هي نظام متكامل لإدارة التوظيف والمهارات، مصمّم ليربط بين ثلاثة أطراف رئيسية: الطلاب الباحثين عن فرص عمل، والشركات المُعلِنة عن وظائف، والجامعات الراعية لطلابها — وكل ذلك تحت إشراف مسؤول النظام.

تدعم المنصة اللغة العربية بالكامل كلغة أساسية مع دعم اتجاه النص من اليمين لليسار، كما تتيح التبديل بين الوضع الفاتح والداكن وتدعم اللغة الإنجليزية كخيار ثانوي.

  • تمكين الطالب من عرض مهاراته والتقدم للوظائف مباشرة
  • تمكين الشركات من نشر فرص العمل وتصفية المتقدمين
  • تمكين الجامعات من متابعة الطلاب المنتسبين إليها

يعتمد المشروع Prisma ORM مع قاعدة بيانات SQLite في بيئة التطوير وPostgreSQL في الإنتاج. فيما يلي الكيانات الثمانية وعلاقاتها:

الكيانالحقول الرئيسيةالعلاقات
User
المستخدم		 id, fullName, email, passwordHash, role, status, refreshToken يرتبط بـ Student أو Company أو University (واحد لكل)
Student
الطالب		 id, userId, universityId, bio, status يتبع User، ينتمي لـ University، يملك مهارات وطلبات توظيف
Company
الشركة		 id, userId, name, industry, status يتبع User، يملك وظائف متعددة
University
الجامعة		 id, userId, name, location, status يتبع User، يضم طلاباً متعددين
Job
الوظيفة		 id, companyId, title, location, type, status, description تتبع Company، تحتاج مهارات، تستقبل طلبات
Skill
المهارة		 id, name, category, status تُربط بالطلاب والوظائف (Many-to-Many)
StudentSkill
مهارات الطالب		 studentId, skillId جدول وسيط (Many-to-Many) بين Student و Skill
Application
طلب التوظيف		 id, studentId, jobId, status يربط Student بـ Job، يمنع التكرار (@@unique)

حماية المسارات في الخادم
// middleware/auth.js — التحقق من الدور
function requireRole(...roles) {
  return (req, res, next) => {
    if (!roles.includes(req.user.role)) {
      return res.status(403).json({ error: "Forbidden" });
    }
    next();
  };
}

نمط اختبار E2E — تسجيل الدخول مع محاكاة الشبكة
// tests/e2e/login.spec.js
test("successful login redirects admin to dashboard", async ({ page }) => {
  // محاكاة استجابة API بدون خادم حقيقي
  await page.route("**/auth/login", route => route.fulfill({
    json: { accessToken: "tok", user: { role: "ADMIN" } }
  }));

  await page.fill("#email", "admin@talenthub.local");
  await page.fill("#password", "Admin1234");
  await page.click('button[type="submit"]');

  // التحقق من التوجيه الصحيح
  await expect(page).toHaveURL(/dashbord\/admin\/dashbord\.html/);
});

الطريقة الأولى: تشغيل سريع بضغطة واحدة (Windows)

الأسهل والأسرع: انقر نقراً مزدوجاً على ملف run-project.bat في جذر المشروع.
سيقوم تلقائياً بتشغيل الخادم الخلفي + خادم الواجهة + فتح المتصفح على صفحة تسجيل الدخول.

الطريقة الثانية: التشغيل اليدوي خطوة بخطوة

الخطوة ١ — تثبيت الحزم (مرة واحدة)
npm --prefix backend install
الخطوة ٢ — إعداد قاعدة البيانات (مرة واحدة)
# نسخ ملف البيئة
copy backend\.env.example backend\.env

# توليد Prisma Client
npm --prefix backend run prisma:generate

# تطبيق مخطط قاعدة البيانات
npm --prefix backend run prisma:push

# زرع بيانات المسؤول الابتدائية
npm run seed:api
الخطوة ٣ — تشغيل الخادم الخلفي
# في نافذة أوامر أولى
npm run dev:api
# الخادم يبدأ على: http://localhost:4000
الخطوة ٤ — تشغيل الواجهة الأمامية
# في نافذة أوامر ثانية
npx http-server -p 5500 -c-1
# الواجهة تبدأ على: http://127.0.0.1:5500
الخطوة ٥ — فتح المتصفح
http://127.0.0.1:5500/login/login.html

النشر عبر Render.com

يحتوي المشروع على ملف render.yaml الجاهز للنشر التلقائي على منصة Render.com (مجانية للمشاريع الصغيرة).

خطوات النشر
  1. أنشئ حساباً مجانياً على render.com
  2. اربط الحساب بمستودع GitHub
  3. اختر "New Web Service" وحدد المستودع
  4. Render سيقرأ render.yaml تلقائياً ويُهيّئ الخدمة
  5. أضف متغيرات البيئة في إعدادات الخدمة (JWT_SECRET، DATABASE_URL، إلخ)
ملف render.yaml
services:
  - type: web
    name: talenthub-api
    env: node
    buildCommand: npm install && npx prisma generate && npx prisma migrate deploy
    startCommand: node src/server.js
    envVars:
      - key: NODE_ENV
        value: production
      - key: DATABASE_URL
        sync: false   # تُدخَل يدوياً في لوحة Render
      - key: JWT_SECRET
        sync: false
إعدادات الإنتاج المطلوبة
