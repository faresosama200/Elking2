خلاصه

شرح أكاديمي مفصل عن جزء الباك اند، أقسامه، ووظيفة كل قسم

هذا المستند مناسب للتسليم الجامعي، ويعرض المفاهيم النظرية مع تطبيقها العملي على مشروع TalentHub.

1) تعريف الباك اند

الباك اند (Back-End) هو الطبقة المسؤولة عن تشغيل منطق النظام في الخلفية. هذه الطبقة لا يتعامل معها المستخدم مباشرة، لكنها تتحكم في كل العمليات الجوهرية مثل استقبال الطلبات، التحقق من صحتها، معالجة البيانات، الاتصال بقاعدة البيانات، حماية النظام، ثم إرجاع الاستجابة المناسبة للواجهة الأمامية.

تعريف مبسط: إذا كان الفرونت اند هو "الواجهة التي يراها المستخدم"، فالـ باك اند هو "العقل الذي يدير النظام".

2) لماذا يعتبر الباك اند عنصرًا أساسيًا؟

  • إدارة البيانات بشكل دائم داخل قاعدة البيانات بدلًا من البيانات الثابتة.
  • تنفيذ منطق الأعمال (Business Rules) مثل شروط التسجيل والصلاحيات.
  • تحقيق الأمان عبر التوثيق والتحكم في الوصول.
  • توحيد مصدر البيانات لجميع الصفحات.
  • تمكين التكامل مع تطبيقات أخرى عبر API.

3) أقسام الباك اند الرئيسية ووظيفة كل قسم

3.1 قسم التوجيه والمسارات Routing

مسؤول عن تحديد مسار كل طلب HTTP وتوجيهه إلى المعالج المناسب. مثال: POST /auth/login يذهب إلى دالة تسجيل الدخول.

الوظيفة: تنظيم الدخول للنظام وضمان أن كل طلب يصل إلى المكان الصحيح.

3.2 قسم منطق الأعمال Business Logic

ينفذ القواعد التشغيلية للنظام: منع التكرار، تطبيق الشروط، تجهيز البيانات قبل الحفظ أو الإرجاع.

الوظيفة: تحويل الطلبات الخام إلى سلوك ذكي يحقق متطلبات المشروع.

3.3 قسم الوصول للبيانات Data Access Layer

يتعامل مع أوامر قاعدة البيانات (إضافة، قراءة، تعديل، حذف) ويعالج العلاقات بين الجداول.

الوظيفة: ضمان تخزين واسترجاع البيانات بدقة وكفاءة.

3.4 قسم التوثيق والتفويض Authentication / Authorization

التوثيق يحدد هوية المستخدم، والتفويض يحدد صلاحياته. مثال: المدير Admin يمكنه إدارة الجداول، بينما مستخدم عادي لا يمكنه ذلك.

الوظيفة: حماية النظام من الوصول غير المصرح به.

3.5 قسم التحقق من المدخلات Validation

يفحص البيانات القادمة من المستخدم قبل تنفيذ أي عملية، مثل التحقق من البريد الإلكتروني وقوة كلمة المرور والحقول الإلزامية.

الوظيفة: الحفاظ على جودة البيانات ومنع الأخطاء قبل وصولها لقاعدة البيانات.

3.6 قسم معالجة الأخطاء والاستجابات Error & Response Handling

ينتج استجابات قياسية مفهومة تتضمن حالة التنفيذ، الرسالة، والبيانات الراجعة، مع أكواد HTTP صحيحة.

الوظيفة: تسهيل التكامل مع الفرونت اند وتحسين التشخيص والصيانة.

3.7 قسم البنية التحتية والتشغيل Runtime & Infrastructure

يشمل بيئة التشغيل (PHP/Apache)، الإعدادات، الاتصال بقاعدة البيانات، والنشر المحلي أو على الخادم.

الوظيفة: إبقاء النظام متاحًا ومستقرًا أثناء التشغيل.

4) تقسيم الباك اند في مشروع TalentHub

يطبق المشروع التقسيم السابق عمليًا عبر وحدات API متخصصة:

القسم الوصف الوظيفي أمثلة عمليات
المصادقة إدارة تسجيل الدخول والخروج والتوكن Login / Register / Logout
الطلاب إدارة بيانات الطلاب وربطهم بالجامعة والمهارات Add / Edit / Delete / View

الجامعات إدارة الجامعات وربطها بالطلاب CRUD كامل

لوحة التحكم عرض المؤشرات والإحصائيات الملخصة Dashboard Stats

5) مفهوم CRUD وأثره في النظام

  1. Create: إضافة سجل جديد إلى قاعدة البيانات.
  2. Read: استرجاع البيانات وعرضها في الجداول.
  3. Update: تعديل سجل قائم بناءً على صلاحيات المستخدم.
  4. Delete: حذف السجل عند الحاجة مع الحفاظ على الاتساق.

وجود CRUD لكل قسم يعني أن النظام انتقل من واجهات ثابتة إلى تطبيق معلوماتي متكامل قائم على البيانات الحية.

6) دورة العمل بين الفرونت اند والباك اند

  1. المستخدم ينفذ إجراء من الواجهة (مثل إضافة طالب).
  2. الواجهة ترسل طلب HTTP إلى API.
  3. الباك اند يتحقق من التوكن والصلاحية.
  4. يتم التحقق من صحة المدخلات.
  5. تنفذ العملية على قاعدة البيانات.
  6. ترجع استجابة منظمة، ثم تحدث الواجهة البيانات المعروضة.

7) الهيكل العام المقترح لمجلدات الباك اند

  • config: إعدادات الاتصال بقاعدة البيانات والمتغيرات البيئية.
  • routes: تعريف المسارات وربطها بالمعالجات.
  • controllers: استقبال الطلبات وبناء الاستجابات.
  • services: منطق الأعمال والقواعد التشغيلية.
  • models / repositories: طبقة الوصول للبيانات.
  • middlewares: التحقق من التوكن، الصلاحيات، ومعالجة الأخطاء.

8) متطلبات الجودة في الباك اند

  • الالتزام بمبدأ فصل المسؤوليات بين الطبقات.
  • توحيد صيغة الاستجابات والأخطاء.
  • حماية كلمات المرور بالتشفير وعدم تخزينها كنص عادي.
  • اختبار endpoints الحساسة (Auth وCRUD).
  • تسجيل الأحداث المهمة لتسهيل المراقبة والتدقيق.

9) خلاصة أكاديمية

الباك اند يمثل البنية المنطقية والأمنية لأي نظام معلومات. نجاحه يعتمد على تصميم طبقات واضحة، وضبط العلاقة بين API وقاعدة البيانات، وتطبيق صارم لمبادئ التحقق والصلاحيات. وفي مشروع TalentHub، تم توظيف هذا المفهوم عبر باك اند PHP/MySQL يخدم تسجيل الدخول، لوحة التحكم، وأقسام الطلاب والشركات والجامعات والوظائف والمهارات بعمليات CRUD كاملة.

إعداد: نسخة عربية بصياغة أكاديمية جاهزة للتسليم.
