تقرير باك اند مشروع TalentHub

نوع المستند: تقرير تسليم جامعي
المجال: Backend Architecture and Implementation
المشروع: TalentHub

1) مقدمة

يهدف مشروع TalentHub إلى ربط الطلاب بالشركات والجامعات من خلال منصة إدارة موحدة. يعتمد نجاح المشروع على وجود باك اند قوي يتولى إدارة البيانات والصلاحيات وتنفيذ العمليات الأساسية لجميع الأقسام داخل النظام.

2) ما هو الباك اند في هذا المشروع؟

الباك اند هو الجزء المسؤول عن استقبال طلبات الواجهة الأمامية، تنفيذ المنطق البرمجي، التحقق من الصلاحيات، التعامل مع قاعدة البيانات MySQL، ثم إعادة النتائج بصيغة JSON.

في TalentHub: الباك اند مبني باستخدام PHP API، وقاعدة البيانات MySQL، مع توكن مصادقة للتحكم في الوصول إلى صفحات الإدارة.

3) الأهداف الوظيفية للباك اند

  • تفعيل تسجيل الدخول وربطه بقاعدة البيانات.
  • السماح بالدخول للمدير فقط إلى لوحة التحكم.
  • جلب بيانات الجداول من قاعدة البيانات بدل الداتا الثابتة.
  • تنفيذ CRUD كامل لكل قسم: Students, Companies, Universities, Jobs, Skills.
  • توحيد الاستجابات وإدارة الأخطاء لتحسين التكامل مع الفرونت اند.

4) أقسام الباك اند ووظيفة كل قسم

4.1 قسم المصادقة (Auth)

يشمل تسجيل الدخول، إنشاء الحساب، تسجيل الخروج، وإنشاء التوكن. يتم التحقق من البريد وكلمة المرور ومطابقة الدور الوظيفي للمستخدم.

4.2 قسم الصلاحيات (Authorization)

يضمن أن العمليات الإدارية لا يمكن تنفيذها إلا بواسطة مستخدم Admin. أي طلب بدون توكن صحيح أو بصلاحيات غير كافية يتم رفضه.

4.3 قسم إدارة الطلاب

مسؤول عن إضافة، عرض، تعديل، حذف بيانات الطلاب وربطهم بالجامعة والمهارات.

4.4 قسم إدارة الشركات

يتولى حفظ بيانات الشركات وتحديثها وحذفها عند الحاجة، ويستخدمها قسم الوظائف لاحقا.

4.5 قسم إدارة الجامعات

يدير بيانات الجامعات المرتبطة بالطلاب، مما يساعد في التصنيف والتقارير.

4.6 قسم إدارة الوظائف

يسمح بإضافة الوظائف وتعديلها وربط كل وظيفة بالشركة المناسبة ونوع الوظيفة.

4.7 قسم إدارة المهارات

يدير قائمة المهارات الأساسية المستخدمة في تصنيف الطلاب وربطهم بالوظائف.

4.8 قسم لوحة التحكم (Dashboard)

يوفر إحصائيات عامة مثل عدد الطلاب والشركات والجامعات والوظائف لمتابعة حالة النظام.

5) قاعدة البيانات المستخدمة

تم اعتماد MySQL كقاعدة بيانات رئيسية للنظام، مع تصميم جداول مترابطة لتغطي جميع الوظائف المطلوبة.

6) معنى CRUD داخل المشروع

  1. Create: إضافة سجل جديد إلى قاعدة البيانات.
  2. Read: عرض البيانات في الجداول داخل صفحات الإدارة.
  3. Update: تعديل البيانات وحفظ التغييرات.
  4. Delete: حذف السجل عند الطلب.

كل قسم من الأقسام الخمسة الأساسية تم تزويده بعمليات CRUD كاملة.

7) تدفق التنفيذ من Login إلى باقي الأقسام

  1. المستخدم يدخل البريد وكلمة المرور في صفحة Login.
  2. يتم إرسال الطلب إلى API للمصادقة.
  3. عند نجاح الدخول يتم حفظ التوكن وبيانات المستخدم.
  4. إذا كان الدور Admin يتم التحويل إلى Dashboard.
  5. من لوحة التحكم يتم الانتقال إلى صفحات الإدارة المختلفة.
  6. كل صفحة تجلب البيانات من MySQL عبر API وتنفذ CRUD عند الضغط على الأزرار.

8) واجهات API المنفذة

  • Auth: login, register, logout, health
  • Students: list, get by id, create, update, delete
  • Companies: list, get by id, create, update, delete
  • Universities: list, get by id, create, update, delete
  • Jobs: list, get by id, create, update, delete
  • Skills: list, get by id, create, update, delete
  • Dashboard: stats endpoint

9) معايير الجودة والأمان

  • التحقق من المدخلات قبل أي عملية حفظ أو تعديل.
  • حماية المسارات الإدارية بالتوكن والصلاحيات.
  • إرجاع رسائل خطأ واضحة عند الفشل.
  • الاعتماد على قاعدة بيانات فعلية بدل بيانات ثابتة.
  • تنظيم بنية API لتسهيل الصيانة والتطوير لاحقا.

10) الخاتمة

تم تنفيذ باك اند متكامل لمشروع TalentHub يحقق المتطلبات المطلوبة: تسجيل الدخول، تحويل المدير إلى لوحة التحكم، وربط جميع الجداول بقاعدة البيانات مع CRUD كامل لكل قسم. هذا يعكس انتقال المشروع من واجهات عرض ثابتة إلى نظام إداري عملي قابل للتطوير.
